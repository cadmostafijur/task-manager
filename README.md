# Task Manager

A simple full-stack task manager web app. Users can add tasks, view them, update status, and delete tasks.

---

## Technologies Used

| Technology | Purpose |
|------------|---------|
| **Next.js 16** | React framework with App Router (frontend + backend in one project) |
| **React 19** | UI library for building interactive components |
| **Next.js API Routes** | Backend REST API (`/api/tasks`) — no separate server needed |
| **Prisma 7** | ORM — talks to the database using JavaScript instead of raw SQL |
| **Neon PostgreSQL** | Cloud-hosted PostgreSQL database |
| **Tailwind CSS 4** | Utility-first CSS for styling |
| **@prisma/adapter-pg** | Connects Prisma to PostgreSQL via the `pg` driver |

### What each folder does

| Folder / File | What it is |
|---------------|------------|
| `app/` | Next.js entry point. `page.js` is the homepage. `api/` holds backend routes. |
| `app/api/tasks/route.js` | **List** (`GET`) and **create** (`POST`) tasks. |
| `app/api/tasks/[id]/route.js` | **Update status** (`PATCH`) and **delete** (`DELETE`) a task by ID. |
| `components/` | React components used by the homepage. Kept outside `app/` for clarity. |
| `lib/prisma.js` | Single shared database connection used by all API routes. |
| `lib/task-status.js` | Status options: To Do, In Progress, Done. |
| `prisma/schema.prisma` | Defines the `Task` table structure in the database. |
| `prisma/migrations/` | Version-controlled SQL that creates/updates tables on Neon. |
| `generated/prisma/` | Code generated by `prisma generate` — imports `PrismaClient` from here. |
| `.env` | Your Neon database URL. Copy from `.env.example` and fill in your values. |

## How to Run

### 1. Install dependencies

```bash
npm install
```

### 2. Set up environment variables

Copy the example file and add your Neon connection string:

```bash
cp .env.example .env
```

Edit `.env`:

```env
DATABASE_URL="postgresql://USER:PASSWORD@HOST/DATABASE?sslmode=require"
```

Get this URL from your Neon project dashboard → **Connection details**.

### 3. Run database migrations

This creates the `Task` table in your Neon database:

```bash
npx prisma migrate deploy
```

### 4. Start the development server

```bash
npm run dev
```

Open **[http://localhost:3000](http://localhost:3000)** in your browser.

### 5. Production build (optional)

```bash
npm run build
npm start
```

## Features

- Add a task with **title**, **description**, and **status**
- View all tasks in a list
- Update status: **To Do** → **In Progress** → **Done**
- Delete a task

## API Endpoints

| Method | Endpoint | Body | Description |
|--------|----------|------|-------------|
| `GET` | `/api/tasks` | — | List all tasks |
| `POST` | `/api/tasks` | `{ title, description?, status? }` | Create a task |
| `PATCH` | `/api/tasks/[id]` | `{ status }` | Update task status |
| `DELETE` | `/api/tasks/[id]` | — | Delete a task |

**Status values:** `TODO`, `IN_PROGRESS`, `DONE`

## Data Flow (simple overview)

```
Browser (React)
    ↓ fetch /api/tasks
Next.js API Routes (app/api/)
    ↓ prisma.task.*
lib/prisma.js
    ↓ SQL via adapter
Neon PostgreSQL
```

1. User interacts with `AddTaskForm` or `TaskList` in the browser.
2. Components call `/api/tasks` endpoints.
3. API routes use `lib/prisma.js` to read/write the database.
4. Prisma sends queries to Neon PostgreSQL.
5. JSON response is returned and the UI updates.
